Here's a fun interview question. Imagine you're running Qwen3.6-35B-A3B at
batch size $1$, on a single GPU, and you decide want to increase throughput.
For whatever reason, your engine can only work on $4$ tokens at a time. You
have two choices[>1]: :

[>1]: Assume you don't have to prefill anything to batch, and that the
sequences are short enough that the KV cache movement doesn't come into play.

1. Run at batch size $4$ by batching 4 random user sequences together.
2. Running at batch size $1$, speculate out to draft length $4$, with
   acceptance length $\alpha$. 

Which is better, assuming you only care about the number of tokens being output
per second (not TPOT, i.e. you don't care about tokens per user per second)? In
a perfectly implemented inference engine, which gives you the maximum output
throughput?

Here's a sensible answer. 

> Assuming that everything is memory bound, batching is always better for
> $\alpha<1$, because there's no chance that a token added to the working set
> by increasing the batch size will be rejected. You'd only choose speculative
> decoding in this case if you wanted to improve the TPOT at the expense of
> some global throughput. At $\alpha=1$, they have the same throughput.

Here's something that's true sometimes:

> Speculating 4 positions out produces, globally, more output tokens per
> second, than running on a batch size of 4, even with $\alpha=0.9$.
